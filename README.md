# Аналитическая система (DWH) и Управленческий BI-дашборд для маркетплейса Wildberries

## Цель проекта
Настройка dbt и создания BI-отчета в Power BI для финансового планирования, контроля юнит-экономики и управления логистикой.

**Ключевые бизнес-метрики, выведенные на дашборд:**
* **Выручка и заказы:** Общий объем заказов (142 тыс.) и фактическая выручка (55.75 млн руб.).
* **Финансовый результат:** Валовая прибыль (-18 млн руб.) и рентабельность (-32.5%), подсвечивающие критические проблемы в текущей экономике брендов.
* **Расходы маркетплейса:** Удержание комиссии WB (5 млн руб.) и стоимость логистики (14 млн руб.).
* **Операционная динамика:** Лидеры роста/падения продаж по категориям (`subject`), география доставок (лидер — Москва, 5.4 млн) и топ возвратов для минимизации кассовых разрывов.

## Архитектура проекта и Пайплайн данных
Проект построен по модульному принципу обработки данных:

1. [Сырые выгрузки API WB / Справочники]
2. [Локальное хранилище в PostgreSQL] (Слой очистки и базовой сборки)
3. [ dbt ] (Аналитический слой витрин: Staging -> Marts)
4. [ Power BI ] (Схема «Звезда», DAX-метрики, Интерактивный дашборд)

## Архитектура модели данных в Power BI
Внутри BI-системы реализована оптимизированная схема **«Звезда» (Star Schema)**, связывающая таблицы фактов с общими измерениями:

* **Таблицы фактов:**
  * `Продажи` — выкупленные и оплаченные клиентами товары (`finished_price`, `for_pay`).
  * `Заказы` — намерения клиентов купить товар, воронка продаж (`is_cancel`).
  * `Отчет_Продаж` — бухгалтерский документ реализации для выверки чистой прибыли и логистических затрат.
  * `Поставки` — входящий поток товаров на маркетплейс.
  * `Склад` — ежедневные срезы остатков на складах маркетплейса для оценки дефицита.
* **Таблицы измерений:**
  * `СПР_Номенклатура` — сквозной справочник товаров (артикулы, бренды, предметы).
  * `СПР_Себестоимость` — финансовые затраты на единицу товара для расчета ROI и валовой прибыли.
  * `СПР_ТарифыДоставка` — логистическая сетка маркетплейса.
  * `Calendar` — единая таблица дат для анализа тайм-серверов и динамики изменений.

## Результаты проекта
* Реализован прозрачный контролируемый пайплайн трансформации данных маркетплейса.
* Разработан инструмент для проверки схождения финансовой прибыли продавца на основе сверки отчетов реализации (`v_report_realization`) и себестоимости.
* Обнаружены ключевые точки падения рентабельности селлера (перерасход на логистику и аномально высокие возвраты по отдельным категориям товаров), что позволяет оперативно скорректировать стратегию продаж.
